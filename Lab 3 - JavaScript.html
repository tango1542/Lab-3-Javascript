Essentials

For these pages, focus on the HTML and JavaScript. No need to add CSS/Bootstrap (unless you would like to). You can also write your JavaScript within your web pages.

Part 1: Create a web page with an empty unordered list, a input text box, and button. When the user types text into the input form and clicks the button, the text should be added to the list, as a new list element. When the text has been added to the unordered list, clear the text from the input. 

Part 2: Start with the previous program, and add validation. Ignore the user input it if it is blank; and display an error message if it is too long - more than 50 characters. Use JavaScript.

Part 3: Create a page with names of a couple of movies and their ratings, as a number out of 5. Use a span or other tag to uniquely identify the numerical value of the rating. For example,

<p>Star Wars, <span class="rating">4 stars</span></p>

<p>Back to the Future, <span class="rating">5 stars</span></p>

Write a script to convert the rating, in the class=rating elements, into the equivalent number of star icons. You can use a star glyphicon, or unicode characters (u2605 and u2606 for example, see unicode.html in the example repository), or download a small star image. So your page will look like something like this,

Star Wars, ★★★★

Back to the Future, ★★★★★

Part 4: Write JavaScript for a web page that includes an element that shows the current number of pixels visible in the browser window. You can calculate this by multiplying the screen width by the height. Update these values as the screen is resized.  Hint: how_wide_is_this_window.html in the example repo.

Part 5: Create a web page that lets the user generate random colors. The user should be able to save the colors that they like. For example, the user may be trying to select a color for text, and would like to see how random colors look as text.

Create an element and JavaScript that generates random colors for another component. (Similar example, random-color.html). You should have a nice large component to display the randomly generated color (perhaps an example paragraph of text? or a div block with non-zero height and width?).

Add a save button that adds a row for the current color to a table.

The saved colors should be displayed in a table with two columns: one column with the background color, and one column for the HEX value of that color.  

To submit: A zip file containing the HTML files for each of these sites. As always, make sure your files have descriptive names. OR push to GitHub and send a link to your repo.

Part 2:  Start with this web page. https://github.com/minneapolis-edu/week3-examples/blob/master/lab3-part1.html

Add a JavaScript script to

  • Count the number of &ltp&gt elements and display the number in the paragraph with id="p_count"
  • Increase the font size of all <b>bold</b> elements with class="large". Don't change any bold elements without this class.
  • Use selectors to specifically identify <i>all</i> italic text within unordered lists. Change this italic text to the color of your choice.
  • Copy the text from all span elements with class="copyme". Sort it in reverse order, and concatenate all the strings together.
  • Add a paragraph containing this information to the div tag with id="copyhere" 
  • Add a mouseover and mouseout listener to the 6th list element. Mousing over this element will cause an image of your choice to appear below the list, mouse out will remove it.

Don't modify the HTML! The objective here is to practice using JavaScript selectors to identify elements and lists of elements.

Part 3: In Essentials Part 1 and Part 2, you used user input to build HTML elements. On this page, try entering some HTML characters in the input box. What happens? How could this cause problems for a website? What could you do to account for this? 

To start with, let's decide these characters are not allowed: < > & " and '. (Greater than, less than, ampersand, double quote and single quote).

Add a key listener that displays a warning message of some kind for the user if they type any of the banned characters in the text box. You can decide how annoying you want to be about this. (Examples: an alert box, a message displayed on the page, change the color of the input text box... or something else).
